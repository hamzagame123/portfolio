#### 3.1 Introduction & Problem (45 seconds)

> Smart Image Renamer is an AI-powered tool that transforms cryptic filenames like 'DSC_8472.jpg' into descriptive, searchable names like 'sunset-beach-california.jpg'.

**[Scroll to problem section]**

> **The Problem:** Creative professionalsâ€”photographers, designers, marketersâ€”deal with thousands of image files. They waste hours manually renaming files, or worse, they can't find images because they're buried under meaningless names.

#### 3.2 Design Process & Research (1 minute)

**[Scroll to user research section]**

> My research revealed three key insights:

> **First:** Users are skeptical of AI. They've been burned by tools that make irreversible changes without warning.

> **Second:** File management is emotional. These aren't just filesâ€”they're memories, client work, years of creative output.

> **Third:** The existing solutions were either too technical for average users or too limited in functionality.

**[Scroll to user journey section]**

> This informed my user journey design. Notice how I mapped not just actions, but emotionsâ€”from initial frustration with messy files, through cautious optimism when trying the tool, to final relief when it works.

#### 3.3 Key Design Decisions (1.5 minutes)

**[Scroll to design/features section]**

> Let me walk you through my key design decisions:

**[Point to relevant UI elements as you mention them]**

> **Decision 1: Preview Before Commit**
> Users can see exactly what names the AI will generate BEFORE any files are changed. This builds trust by making AI behavior transparent.

> **Decision 2: One-Click Undo**
> Every rename operation can be undone instantly. This addresses the fear of irreversible changes and encourages experimentation.

> **Decision 3: Smart Detection**
> The tool only renames files with gibberish names. If a file already has a meaningful name like 'team-photo-2024.jpg', it's left alone. Users maintain control.

> **Decision 4: Visual Thumbnails**
> Instead of just showing filenames, I display image thumbnails so users can visually verify that the AI's description matches what they see.

**[Scroll to design challenges section]**

> The biggest UX challenge was **making AI feel trustworthy**. AI is a black boxâ€”users can't see how it "thinks." My solution was radical transparency: showing the AI's confidence level, allowing manual overrides, and never hiding what the tool is doing.

### SMART IMAGE RENAMER-SPECIFIC QUESTIONS

**Q: Why did you choose to work with AI?**
> AI is reshaping how we interact with technology, but most AI tools feel opaque and untrustworthy. I wanted to explore how interaction design can make AI feel safe, transparent, and user-controlled.

**Q: How does the AI actually work?**
> It uses Google's Gemini Vision API to analyze image content and generate descriptive filenames. The AI "sees" the image and describes what's in itâ€”people, places, objects, colors, mood.

**Q: What happens if the AI gets it wrong?**
> Three safeguards: Preview before commit (see suggestions before any changes), one-click undo (reverse any rename instantly), and manual override (edit any suggested name before applying).

**Q: Why a web app instead of a desktop app?**
> Reduced friction. Users can try it immediately without downloading anything. This was strategicâ€”prove the concept works before asking users to install software.

**Q: What about privacy? Are images uploaded to a server?**
> Great question. Images are processed client-side and sent directly to Google's APIâ€”they never touch my servers. The API key is stored locally in the user's browser. I prioritized privacy because these could be personal photos or confidential client work.

**Q: How did you handle the "black box" problem with AI?**
> Transparency at every step. Users see what the AI suggests, can compare old vs. new names, see thumbnails to verify accuracy, and maintain full control over what gets renamed. The AI is a helper, not a decision-maker.

**Q: What were the biggest technical challenges?**
> Handling large batches of files without freezing the UI. I implemented progressive loading so users see results as they're generated rather than waiting for everything to process.

**Q: Why the dark theme?**
> User testing insight: the same functionality in a dark, polished interface was perceived as more professional and trustworthy than in a light, basic UI. For an AI tool handling important files, perceived reliability matters.

**Q: What's next for this project?**
> A desktop Electron app with advanced features: folder watching for automatic renaming of new files, batch operations UI for processing entire directories, and customizable naming templates.
